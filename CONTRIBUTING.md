# Contributing

PDV is held to production-grade implementation and review standards from the beginning of the project.

Before contributing:

1. Read [docs/governance/agent-directives.md](docs/governance/agent-directives.md).
2. Read the relevant tranche in [docs/roadmap/tranches.md](docs/roadmap/tranches.md).
3. Read the architecture contract in [docs/architecture/go-rewrite-design.md](docs/architecture/go-rewrite-design.md).
4. Use [docs/governance/agent-assignment-prompt.md](docs/governance/agent-assignment-prompt.md) when delegating work to agents.

All contributions must satisfy the documented quality gates, verification requirements, and tranche acceptance criteria before they are presented as complete.

## Pull Requests

- Keep changes scoped and intentional.
- Update documentation in the same pull request when behavior, workflow, or repository conventions change.
- Use the pull request template and complete the verification section honestly.
- Do not present partial scope as complete.
